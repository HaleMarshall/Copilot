Alpha — Moonfare Private-Markets Co-Pilot

Complete functional specification of the peer-benchmark frontend — every page, panel, filter, toggle, chart and interaction. Written for Claude Design as a faithful description of the existing build. Colours are deliberately excluded (the only thing omitted) — every layout, control, chart type, label, state and behaviour is described.

One-screen product. The app is a single screen with a fixed chrome and a swappable centre. The user never scrolls a long page; they switch pages (1–9) via a tab bar, and switch the comparison cohort via the left panel. Three persistent zones surround the centre: a top profile bar, a left cohort/peer-group builder, and a right "Ask Alpha" chat pane. There is also an opening Welcome screen.

0 · Global structure & persistent frame

After the Welcome screen, the layout is a vertical stack: a full-width top bar (fixed height ~60px) over a three-column main row that fills the rest of the viewport.

  • Left column — Cohort / peer-group builder. Fixed width ~280px, collapsible to ~40px via a ‹ / › toggle. Vertically scrollable.
  • Centre column — Page navigation tab bar, a page header (eyebrow + title + subtitle), and the chart grid. Flexes to fill remaining width. The grid switches layout per page (see grid layouts below).
  • Right column — "Ask Alpha" chat pane (~360px), collapsible to ~44px. Always present.

0.1 · Welcome screen

  • Centered card: Moonfare logo, eyebrow "Co-Pilot", serif headline "Welcome to Alpha", tagline "Your AI-augmented Private Markets advisor."
  • Two explanatory paragraphs: the live peer-group engine (compare against peers, the platform, house-view model portfolios, or Harvard / Yale / Princeton endowments) and the Ask-Alpha live-Claude note.
  • A boxed governance disclaimer: which figures are real export data vs "modelled, not measured" (PM-target %, investable-wealth €, the model portfolios), performance caveat (GP marks, young vintages not ranked), and "Informational only — not investment advice."
  • Footer line + a primary "Enter Alpha →" button. Clicking it fades out the Welcome card, runs the data bootstrap (parallel fetch of risk profiles, endowments, you, platform, top-decile, cohort, funds, AI look-through) and renders Page 1.

1 · Left panel — Cohort / Peer-Group builder

This is where the user defines who they are compared against. Header: eyebrow "Compare against" + serif label "My Peers", and the collapse toggle. Body is a set of collapsible <details> sections. Every change immediately re-resolves the cohort (POST to the cohort endpoint) and soft-updates the current page's peer-dependent charts in place (the user's own data never moves; only the comparison moves).

1.2 · Cohort behaviours (statistical-integrity rules)

  • Default = no filters → the peer group is the whole platform. The user adds filters explicitly, so each dimension partitions cleanly.
  • Single-select dimensions (Age, Risk): clicking a chip replaces the selection; clicking the active chip clears it.
  • "All" region clears any specific regions; selecting a specific region clears "All".
  • Live n badge (bottom of the panel): Peers: n = X · of 5,500 total. If the engine had to broaden the cohort to stay above its floor, it appends · relaxed: <dims>.
  • Below-floor gate: if a filter combination drops the cohort below the statistical floor, a full-card overlay appears over the centre reading eyebrow "Only for paid viewers" + serif "Contact your sales associate" + a "Loosen filters" button. (The block is the feature — never show garbage.)
  • "Loosen filters" drops the most-specific active filter (one at a time, in priority order) and re-resolves.

2 · Centre — page navigation & the 9 pages

The tab bar lists 9 pages; the active one is highlighted. A page badge sits on the right of the bar. Below it the page header shows eyebrow "Page N", the page title, and a one-line subtitle. The grid renders the page's cards.

Soft-update model (applies to every page). Toggles and cohort changes do not rebuild the page. Charts mutate in place with no re-animation, and crucially the user's own portfolio chart stays visually static — only the thing being compared against moves. Switching pages does a full rebuild.

Page 1 · Overview — "Overall asset allocation"

Layout grid-2x2. A dismissible AI / IT look-through banner sits above the grid (Page 1 only): "Dedicated AI sleeve: X%" vs a pill "Economic IT/AI look-through Y%" + NAV coverage note — surfacing that stated AI understates embedded AI.

Top-left card — Private Markets allocation

  • Horizontal bar chart, 4 bars: You · actual, Peers · actual, Platform · actual, Model · target. "% of investable wealth" (real commitment ÷ modelled wealth). You/Peers/Platform respond to filters; the Model bar is the house-view target for the chosen risk class. Auto-scales the x-axis to fill the card; inline value labels (1 dp, %).

Top-right card — Fill rate

  • Stacked horizontal bars (Filled + Remaining to 100%), 4 rows: "You (of N% target)", "Peers", "Platform average", "Moonfare Model target". Subtitle shows the user's modelled target %. Inline % labels; explicit pre-rounding. Caveat noted: target % is modelled, so fill rate is illustrative.

Bottom-left card — Strategy mix (dual donut)

  • Two donuts side by side: left = "Your portfolio" (fixed, never changes); right = the selected comparison.
  • A master Benchmark selector (dropdown in the card header) drives BOTH this right donut AND the bottom-right Z-chart: Peers Platform avg Model (<your risk profile>) Harvard endowment Yale endowment Princeton endowment H+Y+P aggregate.
  • Changing it soft-updates only the right donut + the Z-chart + their titles/captions; the left "your" donut is untouched.
  • Donuts share a category colour map across the page so a strategy is the same colour in pie and Z-chart; legend lists every slice with its %.

Bottom-right card — Strategy over/under-weight (Z-chart)

  • Horizontal diverging bar chart at a zero line: You minus the selected comparison, per strategy, in percentage points (signed +/- labels).

Page 2 · Region — "Regional distribution"

Layout grid-2x1.

  • Left — Regional donut (US / Europe / Asia / Rest of World). Pill toggle You Peer Platform sets which mix the pie shows.
  • Right — Region Z-chart (You minus comparison, ppt). Pill toggle Peer Platform sets the comparison baseline. Bar colours match the pie categories.
  • No "Model" option here (models define strategy only, not region); a stale Model selection auto-reverts to Peer.

Page 3 · Format — "Investment format"

Layout grid-2x1. Formats: Primary · Secondary · Direct/Co-Invest · Fund of Fund · Evergreen (legacy "Semi-liquid" is renamed to "Evergreen" on read).

  • Left — Format donut, slices sum to 100%. Pill toggle You Peer.
  • Right — Format Z-chart (You minus comparison, ppt). Pill toggle Peer Platform.

Page 4 · Strategy deep-dive

Layout grid-2x1. The most interactive page.

  • Left — Your strategy donut. The currently "picked" strategy slice is offset/highlighted. Clicking any slice sets the picked strategy and re-renders the right panel.
  • Right — Two-column fund comparison for the picked strategy:
    • "YOUR HOLDINGS" — your real funds in that strategy, largest first; each row = issuer logo + fund name + a proportional bar + %.
    • "PEER · TOP FUNDS" / "PLATFORM · TOP FUNDS" — the funds the selected peer group holds most heavily in the same strategy (volume-weighted).
    A header pill toggle Peer Platform swaps the right column's source (re-renders only the right panel; the donut stays).
  • Empty states: "You hold nothing in this strategy yet." / "Peer holdings unavailable for this strategy."

Page 5 · Sector tree-map — "Sector look-through"

Layout grid-2x2 with fixed row heights (tree-maps ~320px, Z-chart fills the rest). Sectors: Tech · AI · Energy · Industrials · Consumer · Financial · Defense · Healthcare · Other.

  • Top-left — Your exposure tree-map (squarified, gapless tiling). Tile size = your share of each sector; each tile shows the sector name + true %. Always your portfolio.
  • Top-right — Comparison tree-map (the selected peer group). A header pill toggle Peer Platform drives both this tree-map's data and the Z-chart. Each tile additionally shows a delta pill "± N ppt" (you vs that group).
  • Bottom (full width) — Sector Z-chart (You minus comparison, ppt).
  • Tiny sectors get a floored tile size so they remain readable, but always display their true %.

Page 6 · Performance benchmark

Layout grid-3x1 with a toolbar row.

  • Toolbar — two pill toggles: cohort source Peers Platform avg, and tier Average Top 10%. Together they set the benchmark label, e.g. "Peer top-10%".
  • Three metric cards: DPI · MOIC · IRR. Each shows: a big hero number (your value, formatted — DPI 2dp, MOIC "x", IRR "%"), a one-line definition, a "X% of [benchmark]" ranking badge, and a vertical bar chart comparing You / Benchmark / Platform.
  • IRR maturity guard: when the peer group / your portfolio is young-vintage (J-curve), the IRR ranking badge is suppressed and a warning shows: "⚠ Too early to rank — most funds are <4yr vintage; IRR is J-curve-distorted and GP-marked. Use DPI (realised)."
  • Footer per card notes "GP-marked, unrealised."

Page 7 · Vintage distribution

Layout grid-1x1. A single grouped vertical bar chart: for each vintage year, a triplet of bars — You / Peer / Platform share of commitments (%). The vintage label set is the union of your years and the peer/platform years, sorted. Reveals whether your pacing leads or lags.

Page 8 · Follower function

Layout grid-1x1. Demonstrates the Moonfare push notification.

  • Two iPhone mockups (lock screen + home screen) with an overlaid Moonfare push notification: header "M · Moonfare · now", a headline, and a body. The notification text is a live, gap-driven peer alert fetched for the current Follow group (e.g. "your follower group just committed to EQT 10, increasing buyout").
  • Side panel: "HOW IT WORKS" (peer-triggered, gap-targeted — fires within minutes when a peer commits to a fund that closes one of your model gaps), the "CURRENT TRIGGER" (the live alert), and the "FOLLOW GROUP" (set via the left panel → Follow: Peer avg / All / Top 10%).

Page 9 · What's next — "Investment opportunities"

Layout grid-1x1; inside, a two-column layout (feed + sticky right rail). Ranked by how well each opportunity closes the gap between your current allocation and your assigned model portfolio.

Left — opportunity feed

  • A toolbar showing how many funds are selected and total committed €, with a "Clear selection" button.
  • Opportunity cards (one per open/waitlist fund), each: hero image, status badge (Open / Waitlist / Evergreen / Featured), an optional "New" badge, a "Closes N ppt gap" badge when it fills an underweight, issuer logo, issuer · strategy, name, description, region + tags + stat chips, minimum investment, and a "Select fund" action (becomes "Adjust €" / "Remove" once selected).

Fund-select modal (opens on Select / Adjust)

  • Fund name, issuer · strategy · minimum. A capital commitment slider (min → 20×min/€2m, stepped by the minimum) with a live € value and a "% of your wealth" read-out.
  • Quick-set buttons: Min · 2× min · 5× min · 1% wealth · 5% wealth.
  • Cancel / "Add to portfolio". A sensible recommended default (~5% of PM target capital, clamped) is pre-filled.

Right — sticky portfolio rail

  • "Your selection": each picked fund (name + € + remove ×) and the total commitment.
  • "Portfolio impact" (live scenario recompute): PM target before → after (with ppt delta), PM capital before → after, two small before/after donuts, and per-strategy allocation deltas for strategies that changed.

3 · Ask Alpha (right-hand chat pane)

  • Header "Ask Alpha · live · Claude CLI" with a collapse toggle and an "α" avatar. The whole pane collapses to a thin rail.
  • Intro / empty state: an "α" avatar, "Ask Alpha anything", a sub-line, and 4 suggestion cards (each fires a preset question): "Why am I underweight Buyout vs my peers?" · "How do I compare to the Balanced model?" · "Which open funds close my biggest gaps?" · "What is my total AI exposure?".
  • Input at the bottom (text field + Send; Enter submits).
  • On send: the user's bubble appears, then a "Alpha is thinking…" indicator with a live seconds counter. The request carries a full snapshot of live UI state (current page, cohort filters, perf tier, follow group, benchmark pick, picked strategy, profile, selected funds, scenario) so the answer is grounded in exactly what's on screen. The reply renders with lightweight markdown (bold, italic, code, dash-bullets) and a distinct styled "Your next move:" sales-CTA line, plus a "Alpha · Ns" timing label.

5 · Cross-cutting interactions

  • Page navigation — tab bar; full rebuild on switch.
  • Soft updates — filter/toggle changes mutate charts in place (no re-animation); the user's own data stays static so only the comparison visibly moves.
  • Collapsible panels — left cohort panel and right chat pane each collapse to a thin rail.
  • Toasts — transient confirmations (rotate, refresh, fund added, PDF progress).
  • Rotate investor — rebinds to another real masked investor and re-bootstraps.
  • ↻ DB — rebuilds the backend cache and re-bootstraps.
  • Window resize — re-renders the current page so charts re-fit.
  • PDF export — builds a full report off-screen (a hidden second app instance renders all 9 pages so the visible app is never disturbed): a branded cover (investor name + date), a Contents + executive-summary page, then one landscape page per dashboard page with a pixel-perfect chart snapshot and an italic per-page narrative blurb. Saved as Moonfare_Alpha_Review_<name>.pdf.
